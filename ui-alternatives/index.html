How they differ

Each direction makes a deliberate tradeoff. They share the same underlying data — but optimize for fundamentally different ways of working with it.

Axis Command Bridge Agent Council Pulse
Primary metaphor Cockpit / terminal Discussion / panel Feed / stories
How agents appear Conviction bars + debate log Named characters with chat bubbles Thumbs up/down with one-liners
Decision presentation Synthesis card with entry/stop/target "The Council's Verdict" hero quote Big neon "SELL" + "Your move"
How you ask follow-ups Right-rail chat input Reply in thread / @mention agent Tap reply on any story card
Visual language Dark + teal + mono · data-dense Cream paper + serif headlines · editorial Black + neon · gradient-rich
Engagement hook Always-on signals · "what changed?" Curiosity · "what did Finn say?" Streaks, badges, swipe rewards
If you build one thing, I'd build Agent Council.

Your current UI buries the most interesting feature — the fact that multiple agents reach a decision. Information density is fine, but the human-feeling element is what makes this product different from a Bloomberg subscription. Naming the agents and making them argue gives the product a personality competitors can't copy.

Command Bridge is the right second move once power users adopt. Pulse is a separate mobile companion app — don't try to make one UI do all three.
